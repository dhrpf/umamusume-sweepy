# AI Rules — Hard Constraints (project-wide)

Each numbered item = one rule.

## Auth & Secrets

1. Never log raw `auth_key`, `viewer_id`, `udid`, `steam_ticket` values to stdout or any file outside `uma_runtime/auth_cache.json`.
2. Never commit `uma_runtime/`, `*.pfx`, Steam keys, or `accounts.json`.
3. `refresh_auth_before_serving()` must run before any game API call — never bypass.

## API Protocol

4. All game endpoints use msgpack body encrypted with AES-CBC; never construct raw HTTP outside `UmaClient.call()`.
5. Scenario routing: scenario 1 = `single_mode/*`, scenario 2 = `single_mode_team/*`, else = `single_mode_free/*`. Never override manually.
6. Retry 205 (per-call transient), 208 (server busy), 214 (res_ver stale) with the built-in logic; never drop a call without one of these codes or `rc==1`.

## Career Loop

7. `CareerRunner.start()` must run on its own thread. Never call `career()` (or `_run`) synchronously from the FastAPI event loop.
8. State recovery: catch `StateRecoveryError`, then call `_fresh_career_state()`; never re-call `strategy.next_decision()` on stale state.
9. `_drain_events()` processes `unchecked_event_array` for the current turn before `next_decision()`. Never skip this.
10. Training failure is identified by comparing pre/post stats within `exec_command` RES — never assume event 1007 alone means failure.

## Data Integrity

11. Static JSON (`data/*.json`) is generated by `career_bot/master_data.py` from `master.mdb` — never edit it by hand.
12. Career logs live at `uma_runtime/<acct>/bot_logs/career_log_<ts>.json` (10-15MB+). Always analyze via `scripts/analyze_career_log.py`; never Read raw.
13. `command_info_array[i].failure_rate` is the game's real fail chance. Bot-injected `_decision_detail.failure_rate` may differ. Trust the game field for validation runs.

## Behavior

14. `dna_sleep`, `dna_gauss`, `dna_uniform` must gate all API-bound waits — never call raw `time.sleep()` on API paths.
15. Dev mode (career loop, hidden UI toggle) is opt-in via `dev_mode=True`. Never default it True on public endpoints.
16. Inject bot diagnostics into game-invisible fields only (`_label`, `_decision_detail`, `_decision_options`). Never expand the wire payload beyond what the server expects.
