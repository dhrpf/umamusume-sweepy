# Anti-Patterns

Failure modes observed in this codebase. Each = a mistake already made once.

## Career Loop

### AP-001: Reordering runner steps
`_run_loop` has 8 steps in contract order. Skipping `_drain_events` → state poisoned. Skipping `_recover_blocked_state` → strategy reads stale `playing_state`.
→ See `decisions/007-runner-chain.md`

### AP-002: Swallowing error codes outside allowlist
`runner.py:378-381` is the definitive absorb list. Adding arbitrary codes (e.g., 9999) → real bugs silently absorbed. 102 and 1503 = "already done", swallowing elsewhere is logic error.
→ See `errors.md`

### AP-003: Calling strategy synchronously from FastAPI
`CareerRunner._run` MUST own its thread. `strategy.next_decision()` called from request handler = event loop blocks, career stalls mid-turn.
→ See `ai-rules.md:7`

### AP-004: Skipping `_drain_events` before `next_decision`
Events in `unchecked_event_array` mutate state. Decision reads stale turns forever.
→ See `ai-rules.md:9`

## API Protocol

### AP-005: Regenerating SID inside retry except after 208
Only 208 warrants regen. `make_sid(viewer_id, udid)` on every retry → game rejects as replay.
→ See `ai-rules.md:2`, `decisions/002-sid-regeneration.md`

### AP-006: Substituting MD5/SHA for sm5
`sm5 = MD5 + binary salt b'co!=Y;(UQCGxJ_n82'`. SHA or raw MD5 fails SID derivation → auth reject.
→ See `decisions/001-msgpack-aes.md`

### AP-007: Constructing raw HTTP outside `UmaClient.call()`
AES key per-call, struct prefix, msgpack body, then encrypt. Re-implementing outside client → wrong header/crypto → 102.
→ See `ai-rules.md:4`

### AP-008: Reading `result_code` from res top-level
It lives in `response["data_headers"]["result_code"]`. Top-level read = KeyError.
→ See `ai-rules.md:4` bullet 4

## Strategy

### AP-009: Fixing URA `fail_percent` bug backwards
`ura.py:612` reads `cmd.get("fail_percent")` but game sends `"failure_rate"` → bot sees 0% fail. Fix must land on runner side, not strategy side. Until then, preserving parity.
→ See `decisions/010-fail-percent-bug-parity.md`

### AP-010: Returning garbage on missing state
`next_decision` MUST raise `StateRecoveryError` when state missing — never return plausible-but-wrong `Decision`. Garbage decision → bot trains wrong stat forever.
→ See `scenarios.md:1`

### AP-011: Caching `get_section(turn)` across turns
Gameplay phase map is per-turn. Caching across turns → mid called late, late called mid.
→ See `scenarios.md:7`

## Data

### AP-012: Hand-editing `data/*.json`
Generated by `master_data.py` from mdb. Hand-edits wiped on next regen, game version may mismatch.
→ See `ai-rules.md:11`

### AP-013: Reading raw career log JSON directly
Logs 10-15MB+. Always use `scripts/analyze_career_log.py`. Raw Read = context bloat.
→ See `ai-rules.md:12`

### AP-014: Not bouncing proc after `generate_master_data.py`
`skills.py`, `races.py`, `scenarios/ura.py` read `data/*.json` at import. Stale data persists across regen unless proc restarted.
→ See `prompts/regen-master-data.md`

### AP-015: Trusting bot-injected `_decision_detail.failure_rate`
Game sends `command_info_array[].failure_rate` = real fail chance. Bot-injected field may differ. Validation must use game field.
→ See `ai-rules.md:13`

## Timing / Behavior

### AP-016: Raw `time.sleep()` on API paths
Use `dna_sleep`, `dna_uniform`, `dna_gauss`, `dna_randint` — per-install seeded profile bypasses bot detection.
→ See `ai-rules.md:14`, `decisions/008-timing-dna.md`

### AP-017: Expanding wire payload beyond what server expects
Inject diagnostics only into game-invisible fields (`_label`, `_decision_detail`, `_decision_options`). Payload expansion → 102.
→ See `ai-rules.md:16`

### AP-018: Default `dev_mode=True` on public endpoints
Dev loop + hidden UI toggle is opt-in. Default-True → public users loop careers, burn TP ungated.
→ See `ai-rules.md:15`

## Runner dispatch

### AP-019: Adding new `Decision.action` without updating runner
Runner dispatches via if/else on string. New action not in runner → hits `else` at `runner.py:360-363` and BREAKS LOOP.
→ See `scenarios.md:2`, `prompts/add-scenario.md`

### AP-020: Omitting `current_turn` or (`command_id` XOR `command_group_id`) from command payload
Both required. Omit either → runner dispatch raises or silently no-ops.
→ See `scenarios.md:3`

## Env / Config

### AP-021: Stale inherited `UMA_MASTER_MDB`
Env override wins over `settings.json`. Stale value → regen against wrong mdb (silent success, wrong data).
→ See `prompts/regen-master-data.md`, `env.md`

### AP-022: Hardcoding `FRIDA_REMOTE` as auto-detect
When FRIDA_REMOTE set, MUST use remote. Auto-detaching → capture-local even when user asked for remote.
→ See `ai-rules.md:13`
